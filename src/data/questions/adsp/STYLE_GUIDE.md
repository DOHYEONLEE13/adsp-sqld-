## 3. 🧩 보기 구성 패턴

### 3.1 4지선다 직접형
```
① {정답}
② {오답 — 인접 개념의 정의}
③ {오답 — 범위 축소/확대}
④ {오답 — 방향 뒤집기 or 허구 용어}
```

### 3.2 보기 그룹형 (가/나/다/…)
- 3~5 개 진술 제시 (가, 나, 다, [라, 마])
- 정답 형태: `(나)` 단독 / `(가), (나), (다)` / `(나), (라), (마)` 등
- **오답 구성:** 인접 조합(하나 포함·하나 제외) 3종 — 단순 "전체" / "하나만" 등
- 예: 48·Q15 답이 `(나)` → 오답 `(가), (나)` / `(나), (다)` / `(가), (나), (다)`

### 3.3 순서 나열형
- 예: SECI 4단계 순서, 데이터마이닝 5단계 순서, ARIMA 절차
- **오답 구성:** 인접한 2단계 swap 3종

### 3.4 계산형
- **오답 구성 원칙:**
  1. 분모를 잘못 잡은 값 (df 오류: n vs n−1 vs n−2)
  2. 공식 계수 오류 (1.5×IQR 대신 0.5·1.0·2.0)
  3. 인접 지표와 혼동 (지지도 vs 신뢰도 vs 향상도)
  4. 독립 가정 무시 등 확률 오계산

## 8. 🧠 오답 선지 생성 체크리스트

신규 문제 생성 시 각 오답이 다음 3개 중 하나 이상에 해당하도록:

- [ ] **인접 개념**: 같은 토픽 내 다른 개념의 정의를 빌려옴 (예: SECI 정답이 '연결화' → 오답에 '외재화·공통화·내면화' 표준 정의)
- [ ] **함정 패턴**: §4 에서 하나 이상 적용 (방향 뒤집기·범위 축소·유사 용어 치환 등)
- [ ] **전형 오계산**: 계산형의 경우 §3.4 원칙 적용

**품질 거부 조건:**
- 오답이 너무 터무니없어서 바로 제외 가능 → 재작성
- 정답과 오답의 길이·문장 구조가 극단적으로 다름 → 길이 스타일 맞춤
- 3개 오답 중 2개 이상이 동일 함정 패턴 → 패턴 다양화
