> **이 문서의 역할.** "결제 받기 / 사업자 등록 / 세금 처리" 관련해서 지금까지 정리한 모든 내용을 한 곳에 모아둔 노트. 사용자가 "이거 진행해" 라고 말하면 이 문서를 다시 펼쳐서 단계별로 안내.
>

### 손익 분기

## 2. 옵션 B 진행 시 — 사업자 등록 단계별 체크리스트

### 2-1. 홈택스 사업자등록 신청

**경로**: 홈택스 로그인 → 신청/제출 → 사업자등록 신청 (개인)

**핵심 입력값**:
| 항목 | 값 |
|---|---|
| 상호 | QuestDP (또는 본인 명의 한글 상호) |
| 사업장 주소 | 자택 가능 (재택 사업자) |
| 업태 | 도매 및 소매업 / 정보통신업 |
| 종목 | 디지털 콘텐츠 판매 / 정보 서비스 운영 |
| 주업종 코드 | **`525101`** 전자상거래 소매업 |
| 부업종 코드 | **`749609`** 기타 정보서비스업 |
| 과세 유형 | **간이과세자** 로 시작 (연매출 8000만원 미만 자동 해당) |

**업종 코드 — 2026-04-28 사용자 검증 결과 (이전 추천 폐기)**:

QuestDP 처럼 디지털 학습 콘텐츠 구독 (1인 SaaS) 의 한국 실무 표준 조합은:

- **주업종 `525101` 전자상거래 소매업**
  - 디지털 콘텐츠를 일반 소비자에게 직판 (구독·이용권 판매)
  - 토스페이먼츠 PG 심사 시 "디지털 콘텐츠" 카테고리로 통과 깔끔
- **부업종 `749609` 기타 정보서비스업**
  - DB 운영·학습 데이터·문제풀이 시스템 = 정보 서비스 본질

이 조합이 맞는 이유:
- 학원 등록 의무 없음, 게임물 등급심의 의무 없음
- 면세 코드 아님 → 부가세 정상 신고 → 토스 정산 명료
- 외부 PG·세무서 검토 시 모델 일치도 높음

**⚠️ 추천하지 말 것 — 자주 헷갈리는 함정 코드**:

| 코드 | 실제 의미 | 왜 부적합 |
|---|---|---|
| `5821x` (`58211`/`58212`/`58219` 등) | **게임 소프트웨어 개발·공급** | QuestDP 는 게임 아님. PG 심사 시 게임물관리위원회 등급분류 요구 가능성 + 게임산업진흥법 적용 리스크 |
| `8550xx` (예: `855010`) | **학원/교습소** (면세) | 교육청 학원 등록 (시설·강사 요건) 전제. 미등록 사용 시 세무 리스크 |
| `5829x` (응용 소프트웨어) | 패키지 SW 개발·공급 | 구독 콘텐츠 사업과 결이 다름. SaaS 콘텐츠는 525101 + 749609 가 더 자연스러움 |

**⚠️ 코드는 항상 홈택스 직접 검색**

KSIC (한국표준산업분류) 와 국세청 업종코드는 별개 체계이고 주기적 (2017, 2022, ...) 개정. 외부에서 외운 번호는 부정확하기 쉬움.

```
홈택스 → 조회/발급 → 기타 조회 → 업종코드 조회
  키워드 "전자상거래 소매" → 525101 확인
  키워드 "정보 서비스"    → 749609 확인
```

복수 업종 등록 가능 (수수료 0원). 향후 변경·추가도 홈택스에서 즉시.

**소요 시간**: 신청 즉시~3일. 사업자등록증 PDF 발급.

### 2-2. 통신판매업 신고

**경로**: 정부24 (https://www.gov.kr) → "통신판매업 신고"

- 온라인 유료 결제는 거의 필수
- 연 거래 50건 미만이면 면제 신청 가능 — 그러나 토스/스트라이프는 보통 요구함
- 신고증 발급: 즉시~3일

### 2-3. 사업용 통장 + 카드 개설

**중요 원칙**: 사업으로 들어오는 돈·나가는 돈을 개인 가계와 100% 섞이지 않게.

**왜 중요한가**:
```
사업소득 = 매출 - 필요경비
```
개인 카드/통장으로 사업 비용 결제하면 종소세 신고 시 "사업용이라는 증거 없음" → 경비 인정 못 받음 → 사업소득 부풀려져 세금 더 냄.

**셋업**:

| 분리 항목 | 어떻게 |
|---|---|
| 사업용 통장 | 카카오뱅크 사업자 / 토스뱅크 사업자 / 신한·국민 사업자 — 비대면 OK. 토스 정산금이 들어오는 단 하나의 통장 |
| 사업용 카드 | 사업용 통장 연결 체크/신용카드 1장. 모든 사업 결제 (서버비·SaaS·광고비) 는 이 카드만 |
| 매월 사장 본인 월급 | 일정액만 개인 통장으로 이체 (이게 "사장 인건비") |
| (선택) 사업용 메일 | `billing@questdp.dev` 같은 메일로 모든 청구서 수신 |

**1년 뒤 효과**:
- 분리 X: 1년치 카드 명세서 1500줄 분류 → 며칠 + 인정 경비 60~70%
- 분리 O: 사업용 카드 명세서 = 그대로 100% 경비

### 2-4. 토스페이먼츠 가맹점 신청

**올바른 카테고리 선택** (이커머스 아님!):
```
□ 쇼핑몰              ← 여기 X
□ 디지털 콘텐츠        ← QuestDP 1순위
□ 교육·강의            ← 또는 여기
```

**제출 서류**:
- 사업자등록증
- 통신판매업 신고증
- 사업용 통장 사본
- 대표자 신분증
- 홈페이지 URL (`questdp.dev` 등) + 화면 캡처
- 이용약관 / 개인정보처리방침 / 환불 정책 (사이트에 게시되어 있어야 심사 통과)

**심사 기간**: 1~2주

## 3. 세금 캘린더 — 매년 해야 할 일

### A. 부가가치세 (VAT)

| 과세유형 | 신고 시점 |
|---|---|
| 간이과세자 (연매출 8000만 미만) | 1월 25일 — 전년 1~12월분 (1년 1회) |
| 일반과세자 | 1월 25일 (전년 7~12월) + 7월 25일 (1~6월) |

```
매출세액 (받은 부가세) - 매입세액 (낸 부가세) = 납부할 세금
```

토스페이먼츠는 **자동으로 매출 데이터를 국세청에 연동** → 누락 신고 위험 없음. 토스 정산내역서가 신고 자료.

### B. 종합소득세 — 5월 1~31일

전년 1~12월 사업소득 합산. 신고는 매년 5월.

```
매출 - 필요경비 = 사업소득
사업소득 - 인적공제 - 기본공제 = 과세표준
과세표준 × 세율 - 누진공제 = 종합소득세
+ 지방소득세 10%
```

**세율**:
| 과세표준 | 세율 |
|---|---|
| ~1400만원 | 6% |
| ~5000만원 | 15% |
| ~8800만원 | 24% |
| ~1.5억 | 35% |
| 그 위 | 38~45% |

### C. 필요경비 인정 항목 (사업용 카드로 결제 시 자동 인정)

| 항목 | 예시 |
|---|---|
| 서버 / 호스팅 | Supabase, Cloudflare, Mux, AWS |
| 도메인 | Cloudflare Registrar |
| 결제 수수료 | Toss 2.7%, Stripe 등 |
| 외주비 | 디자이너·번역가 (3.3% 원천징수 후 지급 + 신고) |
| 광고비 | 메타·구글 광고 |
| 통신비 / 인터넷 | 사업용 안분 (보통 50%) |
| 사무용품 / 도서 | 모니터·키보드·전공서적 |
| 임대료 (재택) | 작업 공간 비율로 안분 |

## 4. 회계 처리 — 단계별 추천

| 매출 단계 | 추천 방식 | 비용 |
|---|---|---|
| 월 100만원 미만 | 홈택스 직접 신고 (모두채움 신고서) | 0원 |
| 월 100~1000만원 | 회계 SaaS — **자비스 (jobis.co)** / 캐시노트 / 세무톡 | 월 1~3만원 |
| 월 1000만원 이상 | 세무사 위탁 | 월 10~20만원 + 신고 시 30~50만원 |

**QuestDP 시작 단계** → **자비스** 가장 적합. 토스 매출·사업용 카드 명세서·홈택스 자료 자동 연동.

⚠️ 자비스는 SaaS·구독 사업자 지원. **삼쩜삼 / 캐시노트** 일부 화면이 이커머스 셀러 특화라 4가지 카테고리 (전자상거래 소매업, 소매중개업, SNS 마켓, 해외직구대행) 만 보일 수 있음 — QuestDP는 **그 4개 모두 해당 X**. "기타" 또는 "해당없음" 찾거나 다른 SaaS 사용.

## 5. 자동화 — 만들어 둔 것

### `npm run report` (이미 구현됨)

`scripts/monthly-report.mjs` — Supabase 데이터를 매월 Excel 보고서로 자동 생성.

**시트**:
1. 요약 KPI
2. 일별 활동
3. 사용자 TOP 50
4. 학습 콘텐츠 (챕터별 정답률)
5. 어려운 문항 TOP 30
6. 재무 (placeholder — 결제 연동 후 자동 채워짐)
7. 세무 (placeholder — 부가세/종소세 양식)

**셋업**:
```bash
# .env.local 에 service_role 키 추가
echo "SUPABASE_SERVICE_ROLE_KEY=eyJ..." > .env.local

# 실행
npm run report                  # 전월 자동
npm run report -- 2026-04       # 특정 월
```

### 결제 연동 후 추가될 자동화

- Toss Payments API → 시트 6 (재무) 자동 채움
- Gmail MCP → 청구서 메일 라벨 자동 분류 → 매입 자료 시트 자동 채움
- Notion MCP → 월간 대시보드 동기화
- 분기말 자동 부가세 신고서 PDF 생성
